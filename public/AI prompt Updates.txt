IPC Pattern (React → SQLite)
React Component:
  const res = await window.api.products.getAll({ filters })
  if (!res.ok) throw new Error(res.error)
  const data = res.data

Main Process (handlers.js):
  ipcMain.handle('products:getAll', (event, filters) => {
    return db.prepare(...).all()  // Raw data, auto-wrapped as { ok, data }
  })

Dev Fallback Pattern
All components that call window.api must check if (window.api) and provide dummy data fallback for browser dev mode.

📝 CODING STANDARDS (MANDATORY)
Functional components only — no class components
Async/await — no .then() chains
Error handling: if (!res.ok) throw new Error(res.error) on every window.api call
IPC calls must have fallback: if (window.api) { ... } else { /* dummy data */ }
Dummy data at top of file — realistic Kenyan prices, real product names
No hardcoded colors — use Tailwind classes only
Transactions for multi-step writes: db.transaction(() => { ... })()
UUIDs for all IDs: const { v4: uuidv4 } = require('uuid')
Timestamps UTC: datetime('now') in SQLite, format on display in JS
